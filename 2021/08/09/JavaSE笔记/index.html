整理一下javaSE笔记

面向过程和面向对象的比较

面向过程：侧重的是过程

优点：简单
缺点：代码量大了以后维护性很差

面向对象：侧重的是对象

优点：可扩展性非常强，维护成本低
缺点：新手上手难

造车

  1. 画图纸
  • 定义车的属性信息：color、speed、seat…..
  • 定义车的动作：跑
  1. 拿着图纸找工厂生产车

    总结

    面向对象的世界里，类就是图纸，属性就是这一类事物拥有的共同属性，动作就是这一类事物共同执行的功能。而对象就是使用类创建的具体某一个东西。对象能干什么，完全取决于类是如何定义的。

Static静态

静态的内容在内存中是保留一份的，并且各个对象之间进行共享。

静态的特点

1.数据共享
2.属于类的，并不属于对象
3.优先于对象产生的

创建对象的过程

1.静态构造器
2.通用构造器
3.构造方法
4.创建对象
⭐. 这就证明创建对象钱就可以访问静态内容，由于创建对象的过程是在静态内容加载之后完成之后，因此在静态方法和静态块离不能用this。

this与super

关于this与super的作用

this指的是当前类的对象
this可以在方法内部获取到对象中的属性信息
this还可以区分局部变量与成员变量
super表示父类的内容
⭐. 用super和this来区分父类和子类中重名的内容
⭐. 创建对象的过程：先创建父类对象，然后创建子类对象

小总结

  1. super可以获取到父类中的内容
  2. 可以调用父类中的构造方法，必须写在子类构造方法的第一行，如果父类的构造方法无参的，则使用super();。
  3. 如果父类没有无参构造，则需要加参数，super(参数);。

方法的重写

重写的定义：子类对父类中提供的方法进行重新定义
语法：子类和父类中的方法的声明完全一致。

```java
class A {
public void SayHi(){
System.out.println(“Hi”);
}
}
class B extends A {
public void SayHi(){
System.out.println(“Hello”);
}
}
